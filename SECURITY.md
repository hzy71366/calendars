## 报告漏洞

本项目不涉及用户数据存储、认证授权或网络服务，
漏洞风险主要存在于依赖的第三方库（lunar-python、icalendar）。

### 请报告

- 农历日期计算错误（六斋日判定不准确）
- ICS 文件格式错误（导致日历 App 解析失败）
- 脚本执行中的安全风险（如命令注入）

### 报告方式

1. 在 GitHub Issues 提交（**不要**包含敏感信息）
2. 或直接提交 Pull Request 修复

### 响应时间

我们会在 48 小时内回复确认，并在 7 天内给出修复方案。
